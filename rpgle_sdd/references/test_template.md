# 测试用例模板

## 测试用例格式

```markdown
### [TC-XXX] 测试标题

**类型:** POSITIVE / NEGATIVE / BOUNDARY / REGRESSION

**优先级:** P0 (Critical) / P1 (High) / P2 (Medium) / P3 (Low)

**作者:** 
**日期:** 

#### 摘要
简要描述测试目的

#### 前提条件
- 条件1
- 条件2

#### 测试数据
| 字段 | 值 | 说明 |
|------|-----|------|
| FIELD1 | 'TEST001' | 测试用 |

#### 测试步骤
1. 步骤1
2. 步骤2
3. 步骤3

#### 预期结果
期望的系统行为

#### 实际结果
（测试后填写）

#### 状态
✅ PASS / ❌ FAIL / ⏳ PENDING
```

## 常见测试场景

### POSITIVE 测试场景

| 场景 | 测试点 |
|------|--------|
| 正常流程 | 所有字段正确输入 |
| 单条记录 | 新增/修改/删除单条记录 |
| 批量处理 | 按预期处理多条记录 |

### NEGATIVE 测试场景

| 场景 | 测试点 |
|------|--------|
| 必填字段 | 缺失必填字段 |
| 重复数据 | 唯一键重复 |
| 无效数据 | 格式/范围错误 |
| 权限不足 | 无权限操作 |

### BOUNDARY 测试场景

| 场景 | 测试点 |
|------|--------|
| 空值 | NULL/空字符串 |
| 边界值 | 最小/最大值 |
| 超长 | 超过字段长度 |
| 特殊字符 | 全角/特殊符号 |

### REGRESSION 测试场景

| 场景 | 测试点 |
|------|--------|
| 原有功能 | 不受影响的原有逻辑 |
| 数据完整性 | 现有数据不被破坏 |
| 性能 | 响应时间不劣化 |
